### Estructura de la API

- Se implementará un backend que servirá como fuente de datos para múltiples métodos
- Endpoints principales para gestionar el menú del restaurante:
    - GET `/platillos`: Traer todos los platillos disponibles en el menú
    - GET `/platillos/{id}`: Obtener un platillo específico por su número identificador
    - POST `/platillos`: Crear o generar nuevos platillos
- Se implementarán operaciones CRUD completas para los platillos:
    - Actualizar platillos existentes
    - Agregar nuevos platillos
    - Eliminar platillos del sistema

### Ventajas de Flask para el Proyecto

- Framework minimalista que permite construir desde cero
- No incluye estructura base predefinida
- Permite desarrollar completamente el modelo de datos según necesidades específicas
- Facilita la creación de endpoints personalizados
- Permite implementar manejadores de errores personalizados
- Es muy liviano al no venir precargado con componentes innecesarios
